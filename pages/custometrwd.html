1-Min et Max Width

Les propriétés min et max peuvent être utilisées sans le flex, sans le responsive. Mais c'est encore dans ce contexte qu'on les utilise le plus.

min|max-width

Min-Width (à partir de) cette largeur

  • min-width: 640px => on peut utiliser le préfixe sm:mon élément à cibler
  • min-width: 768px; => md
  • min-width: 1024px; => lg
  • min-width: 1280px; => xl
  • min-width: 1536px; => 2xl

Max-Width (jusqu'à) cette largeur

  • max-width: 640px
  • max-width: 768px;
  • max-width: 1024px;
  • max-width: 1280px;
  • max-width: 1536px;
  • A savoir : il n' y a pas de préfix intégré pour remplacé max-width (contrairement à min-width)

Exemple : mx-auto max-w-3xl

Ce texte sera toujours centré (mx-auto) et ne dépassera jamais 768px (max-w-3xl), même sur un écran 4K.

Pour être clair entre le Mobile Forst // Min et Max (width-height) :

  • le préfixe sm: est rigoureusement utilisé pour les requêtes min-width.
  • sm (640px) max-width ne correspond pas à "jusqu'à.
  • Pour cibler l'intervalle de 640px jusqu'à 1023px, vous devez combiner les préfixes sm: (qui allume le style à 640px) et lg: (qui l'éteint à 1024px).

Stratégie Mobile First

Si vous voulez qu'une instruction s'applique jusqu'à 640px (ou autre) et change après, vous devez la définir par défaut (mobile first), puis la remplacer à partir du point de rupture.

exemple : dans div parente : bg-purple-400 sm:bg-yellow-200

Encore + d'instructions

  • w-full : 100% de son conteneur parent (width: 100%). Applique explicitement width: 100%;. C'est l'instruction la plus claire pour dire "prends toute la largeur disponible".
  • w-auto : cette valeur est souvent utilisée dans un contexte Flexbox pour laisser l'élément prendre la largeur nécessaire à son contenu.
  • w-screen, h-screen : 100% de la largeur/hauteur de l'écran (viewport) (width: 100vw)
  • min-w-full : définit la largeur minimale à 100% de son parent : min-width: 100%
  • min-w-min : Définit la largeur minimale selon le contenu minimal : min-content
  • min-w-max : Définit la largeur minimale selon le contenu maximal : max-content
  • min-h-screen : On utilise souvent min-h-screen pour s'assurer que le pied de page (footer) reste en bas de l'écran, même si le contenu de la page est très court.

2-Min et Max Height

Les propriétés min et max height sont utilisées pour conserver une hauteur en toutes circonstances. Selon les contextes on peut limiter une hauteur ou en assurer une minimum.

min|max-height

Min-Height (à partir de) cette hauteur

  • sm (640px) min-height: 640px;
  • md (768px) min-height: 768px;
  • lg (1024px) min-height: 1024px;
  • xl (1280px) min-height: 1280px;
  • 2xl (1536px) min-height: 1536px;

L'objectif de min-height est de garantir qu'une zone reste visible et conserve une certaine taille, même si elle est vide.

Max-Height (jusqu'à) cette hauteur

  • max-height: 640px
  • max-height: 768px;
  • max-height: 1024px;
  • max-height: 1280px;
  • max-height: 1536px;

L'objectif de max-height est de limiter la croissance d'un élément, forçant le contenu à défiler s'il dépasse cette limite.
