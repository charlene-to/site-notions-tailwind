@Apply

Ce qui se trouve dans @apply doit obligatoirement avoir été défini au préalable dans le bloc @theme (ou dans le thème par défaut de Tailwind).

Toute classe utilitaire de Tailwind est le reflet d'une valeur définie dans le theme

Couleur personnalisée : @theme et @apply

  • Je souhaite une couleur personnalisée: accent = #64D297 - vert du theme de ce site
  • Comme cette couleur ne fait pas partie du thème Tailwind par défaut, il faut la crééer dans @theme
  • @theme : --color-accent = #64D297.
  • @apply : peut utiliser en bkg --color-accent dans mon btn-secondary.

Extend

Il est utilisé pour ajouter de nouvelles valeurs (tokens) au Design System de Tailwind sans écraser les utilitaires par défaut.

Le compilateur crée de nouvelles classes utilitaires uniques (Ex: bg-brand-bkg, animate-wiggle)

Diff entre extend et @ apply

  • Je souhaite une couleur personnalisée: blue-primary = #547EB8.
  • Comme cette couleur ne fait pas partie du thème Tailwind par défaut, il faut la crééer dans @theme
  • @theme : --color-blue-primary = #547EB8.
  • @apply : peut utiliser --color-blue-primary dans une class.

Container Queries

Le bloc @container est l'outil qui dit au navigateur d'appliquer un style non pas en fonction de la taille de l'écran entier (@media), mais en fonction de la taille de son parent nommé (voir le h2 en page d'accueil: Concept pour l'architecture des projets Tailwind Du token au css injectée)

Mon h2 va changer de couleur- entre 320 et 767 : en vert

C'est comme une sorte de media-queries mais sur le container

@layer components : Container-name

Avec container-type - name, inline-size, parent, enfant : min et max width

  • Dans input.css >
    @layer components : Contient vos styles réutilisables
  • pour définir une classe personnalisée (ex: .mycontainerQueries)
    qui inclut les propriétés de Container Query (container-type et container-name)
  • Le name est ensuite utilisé comme ancre pour cibler l'élément dans la règle @container mise en dessous.
  • C'est cette classe (.mycontainerQueries, ou .card, etc) qui, une fois dans le HTML, active le nom du container-name pour la règle de requête

Attention : Dans notre exemple, la classe utilitaire du HTML (bg-brand-advanced de notre violet custom ci-dessus) gagne la bataille de la cascade et écrase votre règle conditionnelle background-color: #042a16;. Donc, définissez la couleur par défaut avant la règle conditionnelle.

Clarté des rôles de chacun

Adistinction conceptuelle entre le "Quoi" (la donnée) et le "Comment" (le conteneur)

  • @theme : Créer la donnée
    Définit les valeurs brutes (tokens), les animations (keyframes), les polices et les dégradés
  • extend : Ajoute sans écraser
    À l'intérieur de theme : { extend-} Ajoute de nouvelles valeurs au thème sans supprimer la palette par défaut de Tailwind
  • @layer components: créer mes class
    Dans @layer components :
  • @apply : Assemble et regroupe
    Dans @layer components : Regroupe plusieurs utilitaires atomiques pour former une classe sémantique réutilisable (Ex: .btn-primary).
  • Container queries
  • @layer components : Contient @apply
    Dans @layer components : L'endroit où on crée notre .btn-primary par ex ou .flex-center-all
  • container-name : Ancre une requête
    avec @container : la règle @container ciblera la class que je mettrai dans le html.
  • @container Nom-du-Container-name-donné (min-width: 320px) and (max-width: 767px) : règle du container queries
    toutes les class appelées dedans d'exécuteront dans la fourchette du min et max
  • @layer utilities : Final et Coeur de Tailwind
    Vous ne devez jamais le remplir ; vous l'utilisez juste en tête de votre CSS pour charger le framework
