Mobile First

1-Stratégie Mobile First

Par défaut : en mobile

  • On définit de base le comportement mobile
  • Puis on définit avec le préfixe ex : md: à partir de 768px - je veux le bkg jaune

Pour être clair entre le Mobile First // Min et Max (width-height) :

  • le préfixe sm: est rigoureusement utilisé pour les requêtes min-width.
  • sm (640px) max-width ne correspond pas à "jusqu'à.
  • Pour cibler l'intervalle de 640px jusqu'à 1023px, vous devez combiner les préfixes sm: (qui allume le style à 640px) et lg: (qui l'éteint à 1024px).

Si vous voulez qu'une instruction s'applique jusqu'à 640px (ou autre) et change après, vous devez la définir par défaut (mobile first), puis la remplacer à partir du point de rupture.

exemple : dans div parente : bg-purple-400 sm:bg-yellow-200

Container Queries

Ne pas confondre avec les medias queries - voir cours Advanced

La différence est fondamentale et résume l'évolution du design responsive : les Media Queries sont globales (elles regardent l'écran), tandis que les Container Queries sont locales (elles regardent l'élément).

2-Medias queries

Par défaut : min et max

  • Compilateur Tailwind nous aide
  • On définit avec le préfixe min et max nos plages
  • ex : Appliquer une bordure rouge uniquement aux écrans jusqu'à 700px de large
  • class="max-[700px]:border-red-500"

Le compolateur s'assure que les règles avec la plus grande min-width sont toujours déclarées en dernier dans le fichier CSS final (output.css).

Par exemple, le style lg: est déclaré après le style md:.

Conclusion : Vous n'avez pas à vous soucier de l'ordre manuellement. En utilisant les préfixes de Tailwind (sm:, md:, lg:), vous déclarez simplement vos styles en partant du Mobile, et Tailwind gère la priorité en plaçant les styles des plus grands écrans à la fin du fichier pour vous.

"Last Rule Wins"

Comme en css pure, l'ordre de déclaration est crucial et le principe de la cascade "Last Rule Wins" (La dernière règle gagne) s'applique toujours.

Pour tester un conflit : prouver la règle de la dernière déclaration, vous devez montrer deux déclarations qui se contredisent, avec la même spécificité, et voir laquelle est appliquée.

@media (min-width: 768px) {
.card { background-color: blue; } /* Perdu */
.card { background-color: red; } /* Gagne, car elle est la dernière dans ce bloc */ }
